---

## 🗂️ Format d'une fiche idée

Chaque idée suit **toujours la même structure** (pour pouvoir comparer, filtrer, et plus tard automatiser) :

```markdown
### [Emoji] Nom du concept

**Pitch** : une phrase qui résume l'idée.

**Inspiration / références** : d'où vient l'idée (émission TV, film, jeu existant…).

**Mécanique** : comment ça se joue concrètement, étape par étape.

**Variantes** : déclinaisons possibles.

**Matériel** : ce qu'il faut pour le mettre en place.

**Acte du menu social** : Entrée (Comprendre) / Plat (Vivre) / Dessert (Oser) — où ça colle le mieux.

**Profils sociaux ciblés** : parmi les 6 profils (cf. `../13_PROFILS.md`).

**Statut** : 💡 Idée brute / 🧪 À tester / ✅ Validée en réel.
```

---

## 🔄 Workflow : ajouter une nouvelle idée

À chaque fois que J donne **une nouvelle idée de jeu ou de soirée** :

1. **Lire l'idée** dans le doc / message fourni par J.
2. **La valider contre les 4 principes** de `00_PRINCIPES_EXPERIENCE.md` (grille de validation).
3. **La structurer** au format de fiche ci-dessus (remplir tous les champs ; si un champ est inconnu, mettre "à définir").
4. **L'ajouter** dans `01_BANQUE_IDEES_JEUX.md` à la fin, ou dans la bonne section thématique si elle existe.
5. **Mettre à jour le compteur** d'idées en haut du fichier banque.
6. Si l'idée est **floue ou incomplète**, poser à J les questions manquantes (mécanique précise ? matériel ? acte ciblé ?) plutôt que d'inventer.

## ⚠️ Règles

- **Garder le format de fiche identique** pour toutes les idées.
- **Ne pas inventer de mécanique** : si J n'a pas précisé, marquer "à définir" et demander.
- Une idée reste 💡 **Idée brute** tant qu'elle n'a pas été détaillée ; elle passe 🧪 **À tester** quand la mécanique est complète ; ✅ **Validée** seulement après un vrai test IRL.
- Le **lien avec les sites de `../opportunites/`** est encouragé : si une idée utilise un lieu ou un produit de la banque d'opportunités, le mentionner.
